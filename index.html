The Way of Code: Legalist Edition

1. Code is law. Execution is obedience.

Rules must be unambiguous. Interpretation invites chaos.

constexpr int MAX_USERS = 100;

void addUser(std::vector<User>& users, const User& user) {
    if (users.size() >= MAX_USERS) {
        throw std::runtime_error("User limit exceeded");
    }
    users.push_back(user);
}

2. Favor enforcement over intention.

A system that depends on developer goodwill will fail at scale. Write constraints, not hopes.

class Password {
public:
    Password(const std::string& pwd) {
        if (pwd.length() < 8) throw std::invalid_argument("Too short");
        if (pwd.find("123") != std::string::npos) throw std::invalid_argument("Weak pattern");
        value = pwd;
    }
private:
    std::string value;
};

4. All behavior must be accounted for.

Undefined behavior is disobedience in waiting.

enum class State { START, PROCESS, END };

void handle(State state) {
    switch (state) {
        case State::START: break;
        case State::PROCESS: break;
        case State::END: break;
        default: throw std::logic_error("Unhandled state");
    }
}

5. Human nature is flawed; build for mistrust.

Assume users will break your API. Assume collaborators will misuse your function. Harden your interface.

int safeDivide(int a, int b) {
    if (b == 0) throw std::invalid_argument("Divide by zero");
    return a / b;
}

6. Visibility is control.

Log. Trace. Monitor. You cannot govern what you cannot observe.

void logRequest(const std::string& endpoint) {
    std::cout << "[LOG] Accessed: " << endpoint << " at " << time(nullptr) << std::endl;
}

7. Centralize authority, decentralize execution.

Architecture is the ruler; services are its ministers. Coordination without hierarchy leads to drift.

class ServiceRegistry {
public:
    void registerService(const std::string& name, std::function<void()> handler) {
        services[name] = handler;
    }

    void execute(const std::string& name) {
        services.at(name)();
    }

private:
    std::unordered_map<std::string, std::function<void()>> services;
};

8. Incentives shape the system.

Design feedback loops that reward stability, not cleverness. Align individual contributions with systemic goals.

int stableSort(std::vector<int>& data) {
    std::stable_sort(data.begin(), data.end());
    return data.size();
}

9. Simplicity is strength. Complexity invites subversion.

A complex function hides bugs. A complex process hides blame.

// BAD
int compute(int x, int y, bool flag, int mode, double scale = 1.0) {
    return x + y;
}

// BETTER
int add(int x, int y) {
    return x + y;
}

10. The good engineer enforces discipline. The great engineer codifies it.

Standards are the scaffolding of greatness.

template<typename T>
concept Incrementable = requires(T x) {
    { ++x } -> std::same_as<T&>;
};

template<Incrementable T>
void incrementAll(std::vector<T>& items) {
    for (auto& item : items) ++item;
}
